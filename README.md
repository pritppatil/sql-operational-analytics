# SQL Operational Analytics

## Business Problem
Operations teams require reliable KPI monitoring to track performance, identify trends, and detect anomalies that impact decision-making.

## Dataset
- Source: Sample Superstore Sales Dataset
- Records: ~10,000
- Data includes order details, revenue, profit, region, and category.

## Tools Used
- SQL (CTEs, Aggregations, Joins)
- MySQL / PostgreSQL / SSMS

## Analysis Performed
- KPI computation for revenue, profit, and order volume
- Monthly and regional trend analysis
- Anomaly detection using statistical thresholds

## Key Insights
- Seasonal revenue patterns identified
- High-impact revenue spikes detected
- Regional performance differences highlighted

## Business Value
This analysis supports operational monitoring, proactive issue detection, and data-driven planning.
